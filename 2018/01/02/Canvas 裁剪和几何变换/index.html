Canvas 裁剪和几何变换

2018-01-02 “android” ,“android 进阶” 阅读量 
文章概要：此文章主要是学习 android canvas 的各种操作（裁剪 与 几何变换）

首先我们先自定义一个 View，在里面绘制一个和布局大小一样的 Rect 和一串文字方便我们对 Canvas 进行操作的时候清晰明了，

裁剪

裁剪一共有两种方式 ：clipRect 、clipPath

矩形裁剪

canvas.clipRect(int left, int top, int right, int bottom)

对中间绘制的矩形进行裁剪，画布被裁剪后，绘制的 rect 和 文字只能看到裁剪后的效果。

根据路径裁剪

canvas.clipPath(Path path)

path:裁剪的路径

绘制一个圆形路径，canva 就可以根据这个路径进行图片的裁剪，当然还可以使用其他的路径。具体效果请自己尝试

几何变换

几何变换方式一共有 4 种。分别是：

  • 移动
  • 旋转
  • 缩放
  • 扭曲

我们可以通过 canvas 、matrix、camera 来实现上面几种几何变换。

几何变换依赖于坐标系，一个手机平放在桌面上以当前布局的左上角为坐标（0，0），向右为 x 坐标，向下为 y 坐标。

如图：

canvas 位移

先在布局顶部和左边的边界上画两条线，当做 x、y 轴。不过因为已经画了一个铺满整个布局的 Rect 所以整个边界线
会暂时看不到。画线的目的就是让我们能够看到对 Canvas 进行移动后，整个布局变成什么样子了。

移动画布：canvas.translate(float dx,float dy)

dx：画布横向移动的距离

dy：画布纵向移动的距离

效果：

上图中我把画布横向移动了 canvasW / 4 的长度，在画布移动后我们绘制了一个矩形和一个“矩形”两个字，大家
可能会发现绘制的矩形我们是用布局的宽高来当做矩形的宽高，绘制出来的效果应该是整个布局都是粉色才是。
出现这样的结果就是因为我们使用了 canvas.translate 这个方法，是的画布整体向右移动了。
我们可以想象现在你坐在教室的前方的墙壁（CanvasTransition），在教室的前方左边有一个投影仪用的幕布（相当于 canvas），我们把幕布（canvas）一半移动到教室外面，一半留在里面，这时投影的画面在教室外面的我们是看不到的，就相当 图2的效果。我们在画布旋转后在绘制 rect 和 文字，rect 和 文字的原点不再是画布之前的 （0，0）而是 位移过后的（canvasW / 4, 0）。

canvas 旋转

canvas.rotate(float degrees)

degrees:旋转的角度

canvas.rotate(float degrees, float px, float py)

degrees：旋转角度

px：旋转 x 轴心点

py:旋转 y 轴心点

旋转也是对整个画布进行旋转了 45°，旋转的中心点默认是 View 的左上角(0,0)，所以在画布旋转后在绘制的内容都会旋转 45° 后再进行绘制。

canvas 缩放

canvas.scale(float sx, float sy)

sx: 横向缩放倍数

sy：纵向缩放倍数

canvas.scale(float sx, float sy, float px, float py)

sx: 横向缩放倍数

sy：纵向缩放倍数

px:横向缩放的中心点

py:纵向缩放的中心点

canvas 的缩放也是默认以 View 的左边顶点为缩放的圆点。

canvas 扭曲

canvas.skew(float sx, float sy)

sx: x 轴方向的扭曲程度

sy：y 轴方向的扭曲程度
